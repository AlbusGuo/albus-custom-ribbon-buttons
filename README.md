# Custom Ribbon Buttons

Custom Ribbon Buttons 是一个 [Obsidian](https://obsidian.md) 桌面端社区插件，用于重新组织左侧功能区（Ribbon）的布局，并支持在其中添加完全自定义的功能按钮。

通过本插件，用户可以将 Obsidian 原生散落在底部的库切换、设置、帮助等系统按钮统一整合到左侧功能区；同时，还可以自由创建新的快捷按钮，将常用命令、文件、外部链接一键触达。

## 功能概览

| 功能 | 说明 |
|------|------|
| 整合内置按钮 | 将库切换、设置、帮助等原生按钮调整到左侧功能区统一管理 |
| 隐藏默认功能区 | 可选隐藏 Obsidian 默认的功能区按钮行，使界面更加简洁 |
| 添加自定义按钮 | 创建执行命令、打开文件或跳转网址的快捷按钮 |
| 添加分割线 | 在按钮之间插入视觉分割线，实现分组管理 |
| 拖拽排序 | 在功能区和设置页均支持通过拖拽调整按钮与分割线的排列顺序 |
| 图标切换 | 每个按钮支持配置主图标和切换图标，点击后自动切换显示状态 |
| 自定义 SVG 图标 | 除 Obsidian 内置图标库外，支持导入自定义 SVG 文件作为按钮图标 |

## 系统要求

- **Obsidian 版本**: 1.12.0 或更高
- **运行平台**: 仅限桌面端（Windows / macOS / Linux）

本插件依赖桌面端特有的左侧功能区 API，不支持移动端使用。

## 安装方式

### 通过社区插件市场安装

1. 打开 Obsidian，进入 **设置 -> 社区插件**。
2. 关闭安全模式（如尚未关闭）。
3. 选择 **浏览**，搜索 "Custom Ribbon Buttons"。
4. 选择该插件，点击 **安装**，安装完成后点击 **启用**。

### 手动安装

1. 从 [GitHub Releases](https://github.com/AlbusGuo/albus-custom-ribbon-buttons/releases) 页面下载最新版本的 `main.js`、`manifest.json` 和 `styles.css` 三个文件。
2. 在 Vault 目录下创建文件夹：`<Vault>/.obsidian/plugins/albus-custom-ribbon-buttons/`。
3. 将上述三个文件放入该文件夹。
4. 重启 Obsidian 或重新加载插件列表。
5. 进入 **设置 -> 社区插件**，找到 "Custom Ribbon Buttons" 并启用。

---

## 快速上手

1. 启用插件后，左侧功能区的底部系统按钮（库切换、设置、帮助）将被自动整合到功能区主区域。
2. 进入 **设置 -> Custom Ribbon Buttons**，点击 **添加新按钮**。
3. 为按钮配置名称、图标、动作类型（命令 / 文件 / 网址）及对应的目标值。
4. 保存后，新按钮将立即出现在左侧功能区。
5. 如需调整顺序，可直接在功能区拖拽按钮，或在设置页通过拖拽手柄重新排列。

## 功能详解

### 全局设置

插件提供两个全局开关，位于设置页顶部：

- **调整内置按钮到左侧功能区**: 开启后，Obsidian 原生的库切换、设置、帮助按钮将被隐藏在其原始位置，并在左侧功能区底部重新创建。关闭后恢复原始布局。默认开启。
- **隐藏默认功能区**: 开启后，Obsidian 默认功能区（左侧 Ribbon 的上方操作按钮行）将被完全隐藏。适用于希望仅保留自定义按钮的场景。默认关闭。

### 自定义按钮

每个自定义按钮包含以下配置项：

| 配置项 | 说明 |
|--------|------|
| 按钮名称 | 鼠标悬停时显示的提示文字（Tooltip） |
| 主图标 | 按钮的默认显示图标 |
| 切换图标 | 点击按钮后切换显示的图标（详见"图标切换"一节） |
| 动作类型 | 按钮点击后执行的动作类型，支持三种模式 |
| 目标值 | 根据动作类型填写对应的命令 ID、文件路径或网址 |

**三种动作类型：**

- **命令 (Command)**: 执行 Obsidian 的任意已注册命令。点击目标值输入框后，将弹出命令搜索面板，支持模糊搜索快速定位。
- **文件 (File)**: 在新标签页中打开指定的 Vault 内文件。点击目标值输入框后，将弹出文件搜索面板。
- **网址 (URL)**: 在系统默认浏览器中打开指定的外部网址。

### 分割线

在按钮列表中可插入分割线，用于在功能区中对按钮进行视觉分组。分割线在功能区中呈现为一条水平细线，与按钮共享相同的拖拽排序能力。

在设置页中，点击 **添加分割线** 即可在列表末尾追加一条分割线。

### 拖拽排序

本插件在两个层面支持拖拽排序：

- **功能区拖拽**: 在左侧功能区中，可直接拖拽自定义按钮和分割线来调整它们的排列顺序。
- **设置面板拖拽**: 在插件设置页的按钮列表中，每个条目右侧均有一个拖拽手柄（竖向抓手图标），通过该手柄可对按钮和分割线进行重新排序。

两种拖拽方式均会实时保存排序结果。

### 图标切换

每个自定义按钮支持配置两个图标：**主图标**和**切换图标**。

- 按钮初始显示主图标。
- 每次点击按钮时，图标在主图标和切换图标之间交替切换。
- 切换状态会被持久化保存，重启 Obsidian 后保持上次的显示状态。

该特性适用于需要状态指示的场景，例如为"开启/关闭某功能"类命令配置不同的视觉标识。如果不需要此功能，将主图标和切换图标设置为同一图标即可。

### 自定义图标

除 Obsidian 内置的图标库（Lucide 图标集）外，本插件支持导入自定义 SVG 图标：

1. 在任意按钮的图标选择器中，顶部第一项为 **自定义图标** 选项。
2. 点击后进入自定义图标管理面板，可查看、选择、删除已有的自定义图标，或添加新图标。
3. 点击 **添加新图标**，在弹出的上传对话框中选择一个或多个 `.svg` 文件。文件名（不含扩展名）将作为图标 ID。
4. 导入完成后，自定义图标将出现在图标选择器的列表中，以 "(自定义)" 标注与内置图标区分。

自定义图标以 SVG 源码形式存储在插件配置中，不依赖外部资源。

## 设置项参考

以下为插件设置页的完整配置结构：

```
设置 -> Custom Ribbon Buttons
|
+-- [全局设置]
|   +-- 调整内置按钮到左侧功能区    (开关，默认：开)
|   +-- 隐藏默认功能区              (开关，默认：关)
|
+-- [自定义按钮列表]
|   +-- 按钮
|   |   +-- 名称      (文本输入)
|   |   +-- 主图标    (图标选择器)
|   |   +-- 切换图标  (图标选择器)
|   |   +-- 类型      (下拉选择：命令 / 文件 / 网址)
|   |   +-- 目标值    (文本输入，命令和文件类型支持搜索选择)
|   |   +-- 删除      (按钮)
|   |   +-- 拖拽手柄  (拖拽排序)
|   +-- 分割线
|   |   +-- 删除      (按钮)
|   |   +-- 拖拽手柄  (拖拽排序)
|   +-- ...
|
+-- [添加新按钮]    (按钮)
+-- [添加分割线]    (按钮)
```

## 已知限制

- **仅支持桌面端**。本插件使用了 Obsidian 桌面端特有的左侧功能区内部 API，无法在 iOS 或 Android 上运行。
- **内置按钮不可拖拽**。通过"调整内置按钮到左侧功能区"功能移入的库切换、设置、帮助按钮固定排列在功能区底部，不参与拖拽排序。
- **自定义图标格式**。仅支持 SVG 格式的图标文件。建议使用 24x24 视口尺寸的 SVG 以获得最佳显示效果。
